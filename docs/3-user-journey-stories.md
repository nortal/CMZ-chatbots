# User Journey Stories

This document outlines the primary user journeys for the CMZ Chatbots platform, describing the goals, motivations, and typical interactions for each user type.

---

## 1. Admin

**Goal:**  
Manage user accounts, system configuration, and performance metrics.

**Why:**  
To ensure the platform runs smoothly, securely, and within project goals.

**Journey:**
- Log into the admin dashboard with secure authentication.
- View system health metrics, usage statistics, and conversation logs for all animals.
- Add, update, or deactivate users (Admins, Zookeepers, Parents, Students).
- Configure system-wide settings such as API keys, AI model parameters, and content filters.
- Monitor performance metrics and adjust configuration to optimize response times.
- Review flagged conversations for potential violations or issues.
- Export reports for internal stakeholders.

---

## 2. Zookeeper

**Goal:**  
Monitor questions submitted to the animals and update their knowledge/content.

**Why:**  
To keep the chatbot’s answers accurate, engaging, and aligned with educational goals.

**Journey:**
- Log into the Zookeeper portal.
- Select a specific animal chatbot from the animal list.
- View recent conversation transcripts, with student nicknames anonymized where necessary.
- Flag incorrect or outdated responses for review.
- Update the animal’s knowledge base with new facts, seasonal updates, or event-specific info.
- Add or edit personality/configuration parameters to keep tone consistent.
- Test updates by simulating a conversation in preview mode.
- Publish the updated knowledge set to the live chatbot.

---

## 3. Parent

**Goal:**  
Monitor my child’s conversations and delete all or part of the conversation history.

**Why:**  
To ensure their safety and maintain control over stored information.

**Journey:**
- Log into the parent dashboard with linked student accounts.
- Select a child’s profile from the list.
- View a chronological history of conversations with animal chatbots.
- Filter by date, topic, or animal to review specific interactions.
- Delete specific conversation turns or the entire history.
- Optionally download the conversation history for personal records.
- Set parental controls such as time limits or blocked topics.

---

## 4. Student

**Goal:**  
Chat with an animal using my name or nickname in the system.

**Why:**  
To learn and interact with the animals in a fun, personalized way.

**Journey:**
- Log in or scan a QR code at the zoo to access the animal chat system.
- Choose an animal from the available list.
- Greet the animal and receive a friendly, personalized introduction.
- Ask questions or respond to prompts from the animal.
- Receive responses tailored to age, nickname, and prior interactions.
- Optionally receive educational follow-up questions or activities from the animal.
- End the session and optionally see a “Today You Learned” summary.
